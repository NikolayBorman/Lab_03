## Проблема 2 
**Дано:**  
IP-адрес узла: 10.101.99.228  
Исходная маска: 255.0.0.0  
Новая маска: 255.255.128.0  
  
10.101.99.228/8 /17  255.255.128.0  

00001010.01100101.01100011.11100100  

11111111.11111111.10000000.00000000  

2^9 = 512 new networks   
2^15 - 2 = 32766 hosts  

**Диапазон номеров подсетей :**   

00000000 0 – 0.0  
00000000 1 – 0.128  
...  
11111111 0 – 255.0  
11111111 1 – 255.128

**Для 10.101.99.228:**  
второй октет = 101 (01100101)  
первый бит третьего октета = 0 (т.к. 99 = 01100011, старший бит = 0)  
→ номер подсети = 101.0  
→ сетевой адрес: 10.101.0.0/17

00001010.01100101.00000000.00000000   – сетевой адрес  
00001010.01100101.01111111.11111111   – широковещательный

10.101.0.0/17      – network  
10.101.0.1/17      – 1st  
10.101.127.254/17  – last  
10.101.127.255/17  – br


| Параметр | Значение |
|----------|----------|
| Количество бит подсети | 9 |
| Количество созданных подсетей | 512 |
| Количество бит узлов в подсети | 15 |
| Количество узлов в подсети | 32766 |
| Сетевой адрес этой подсети | 10.101.0.0 |
| IPv4-адрес первого узла | 10.101.0.1 |
| IPv4-адрес последнего узла | 10.101.127.254 |
| Широковещательный IPv4-адрес | 10.101.127.255 |
